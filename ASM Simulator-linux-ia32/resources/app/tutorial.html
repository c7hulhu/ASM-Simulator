Tutorial

Step 1: Setting Up Your Simulation

Before you start priming and recording information, click the Parameters button. Inside Parameters, you will see the default settings. Change them as suits the desired simulation.

Number of Attributes per Meaning/Spelling
The number of nodes in the brain that correspond to each spelling of the homophone - e.g., if it was set to 50, "nun" would have 50 nodes and "none" would have 50 separate nodes, for a total of 100 nodes.

Priming Method: Word, Phrase, or Picture
The impact of priming dominant or secondary meaning. Priming based on a picture will cause the selected nodes' weights to increase more than priming based on a word or phrase.

Context Switch Resistance Level
The minimum ratio between dominant and secondary meaning that must occur for a word's meaning to be chosen. If set to 75, the dominant meaning can only be chosen when the overall weight ratio between dominant and secondary meaning is 75:25.

Attribute Weight Representation
What shows up in the node boxes on the front page - a decimal, or a percentage.

Baseline Attribute Weight Distribution
If set to random, the nodes' values will be randomized, while maintaining the desired context switch resistance level. if set to normal, all values will be uniform while maintaining the ratio.

Step 2: Running Your Simulation

All data is presented on-screen, without the need to read from a file. Simply press either the primeDominant or primeSecondary button to prime a meaning. Your sequence will appear below the right box. When the context switch resistance level is crossed, the application will notify you.
